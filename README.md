# Baby Tracker — Telegram Mini App

Семейное приложение для отслеживания кормлений, сна, визитов к врачу, анализов, документов и памятных моментов.

## Архитектура

- **Frontend:** статический SPA на GitHub Pages, открывается как Telegram Mini App
- **Backend:** Supabase — PostgreSQL (+ Row Level Security), Storage, Edge Functions, pg_cron
- **Уведомления:** Edge Function по крону каждые 5 минут находит просроченные напоминания и шлёт сообщения через Telegram Bot API в личку каждому участнику с включёнными уведомлениями

## Модель доступа

| Роль   | Инфо-страница | Кормления/сон/визиты/файлы | Участники |
|--------|---------------|----------------------------|-----------|
| admin  | чтение/запись | чтение/запись              | управление |
| editor | чтение/запись | чтение/запись              | просмотр   |
| guest  | только чтение | нет доступа                | нет        |

Аутентификация: Mini App передаёт `initData` в Edge Function `auth-telegram`,
та проверяет HMAC-подпись Telegram, находит пользователя в whitelist-таблице
`members` и выдаёт JWT с claim `app_role`. Все политики RLS опираются на этот
claim — доступ ограничен на уровне базы, а не интерфейса.

## Напоминания

- **Кормления** — интервальные: «напомнить через N минут после последнего
  кормления», с тихими часами (`feeding_reminder_settings`)
- **Визиты и произвольные** — разовые записи в `reminders` с `fire_at`

## Структура репозитория

```
babyapp/
├── supabase/
│   └── migrations/
│       └── 001_init.sql      # схема, RLS, гранты (протестировано)
├── test/
│   └── rls_test.sql          # тестовый стенд: эмуляция Supabase + 13 проверок
└── (далее)
    ├── supabase/functions/   # auth-telegram, send-reminders
    └── web/                  # Mini App (SPA)
```

## Запуск тестов локально

Требуется PostgreSQL 15+:

```bash
sudo -u postgres psql -v ON_ERROR_STOP=1 -f test/rls_test.sql
```

Стенд создаёт БД `babytest`, эмулирует `auth.jwt()` и роли Supabase,
применяет миграцию и проверяет сценарии guest / editor / admin / anon
плюс constraint-ы целостности (ветки смешанного кормления и т.д.).

## Дорожная карта

1. ✅ **Фаза 0** — схема БД + RLS + тесты
2. **Фаза 1** — Edge Functions: `auth-telegram` (initData → JWT), каркас Mini App, главный экран «кормление в два тапа»
3. **Фаза 2** — сон, подгузники, вес/рост; инфо-страница с графиками; гостевой режим
4. **Фаза 3** — визиты + напоминания (`send-reminders` на pg_cron), интервальные напоминания о кормлении
5. **Фаза 4** — файлы: анализы и документы (Supabase Storage, сжатие фото на клиенте)
6. **Фаза 5** — «на память»: лента фото и историй
